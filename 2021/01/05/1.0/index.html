然后第一篇博客就记录一下搭建过程、遇到的问题以及解决方案吧。

那么在依次安装了Git、nodejs和typora之后（这里特别感谢学长把typora的安装包发出来，官网下载速度感人），就要开始正式的搭建了。

第一步进入Git base

执行了测试命令：

git version

node -v

npm -v

得到了与教材相同的返回之后，根据教程开始安装博客。

当时看到了“INFO Start blogging with Hexo!” 认为hexo已经配置成功

同时注意到了后续的命令返回的“warn”是报错，但第一次接触没有解决思路。

所以选择继续跟教程走。

执行命令：

返回：

这是开启博客服务的命令，由于教程上没有展示正确开启服务后的返回截图。
我认为此时博客已正常开启服务……

所以进入浏览器输入了“http：//localhost：4000”，浏览器返回localhost已拒绝连接。

由于没有认识到真正问题是hexo没有开启服务，我开始在网上搜索”localhost已拒绝连接“的解决方案，大致得到以下两种：

1. 4000端口被占用，我根据解决方案 在cmd里执行netstat 查看端口情况
4000端口并无异常，此条遂pass。

2. windows功能中的“Internet Information Service”及“FTP服务器”未开启
按图索骥，发现这几项确实没有勾选，打勾开启服务之后，再次访问博客地址
依旧返回”localhost已拒绝连接“此条也pass。

在搜索中，发现搜索结果中返回了大量的tomcat相关问题，遂在“localhost已拒绝连接”后加“hexo”以提升搜索精确度。

这次返回了hexo官网的“服务器”这一页，学习了相关的server命令，但没有解决拒绝访问的问题。

期间搜索了git中的其他返回语句

此时我又执行了一次

获得结果为“报错是这样的,这个其实是无所谓的,不用管他都可以,”

ok 跳过这个错误之后再次尝试开启服务，依旧返回熟悉的：

这次，在近乎绝望中，我在网上搜索了“Usage: hexo ”

对于命令提示符玩的不怎么熟练，也没学过linux的我，照葫芦画瓢也没画出个所以然。（此处参考博客：https://www.jianshu.com/p/2f4aca924fdb和https://www.jianshu.com/p/3a971e8c1209）

执行cd d： 看到末尾出现了“/d”心情突然兴奋。

依次执行了 cd Git 和 cd myBlog（前两次画瓢画失败，让我心灰意冷，干脆一步一步来）

终于到了blog路径下！

接下来就是部署到Github Page了 。

在Github中的操作

此时已经是次日的1：33了 ，跟着教程无论是注册Github的账号还是创建一个新的存储库，都十分顺利。

随后的查询、配置ssh、给GitHub pages分配公匙也都操作成功。

万事俱备，只欠一句

hexo d

然后无论我怎么部署，浏览器中显示的只有那句创建HTML页面时打的“Hello World”

本着对自己生命健康财产安全负责的态度，我老老实实的选择上床睡觉。

一觉醒来，继续部署，半个小时过去没有一点进展。

在bing上搜索“hexo INFO Validating config错误”找到了一篇博文，解决了我的问题。

（http://smartadmin.com.cn/smartadmin/show-3955.html）

原文如下：“原来从2020.10.1开始github默认分支从master改为了main，教程及脚本都还是默认master，所以只需要将_config.yml文件中改为部署到github的main分支即可，然后打开github.io主页，成功打开。”

照做，果然成功。

这一番折腾之后，我终于在网上看到了由GitHub Pages 静态站点托管服务呈现的我的hexo模板博客。
